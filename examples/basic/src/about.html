DomPile

Peaceful rebellion through simplicity.

Our Purpose

DomPile is a minimalist CLI tool that empowers developers to build static HTML pages using includes and head injection—no JavaScript frameworks or complex build systems required.

Manifesto:
Reclaim the DOM. Pile your includes, not your dependencies.

Inspired in part by early web tools like the Dogpile search engine, DomPile carries forward the spirit of peaceful aggregation—only here, you assemble your pages from HTML bits with clarity and intention.

The Principles Behind DomPile

Simplicity

Just HTML includes and head injection—no runtime complexity.

Peaceful Assembly

Your right to assemble pages bit-by-bit, calmly and predictably.

Modern Standards

Native ESM support; works on Node, Bun, Deno; supports build & serve, dry‑run, verbose modes.

Eco‑Tech Aesthetic

Solarpunk-inspired visuals powered by Augmented‑UI CSS for calm futurism.

How It Works

$ dompile build
Processing: index.html → dist/index.html
Injecting includes into head…
Build complete in 112 ms

Run dompile build --input src --output dist to generate your site. Includes are resolved recursively, head injections applied, and output written to dist/. Use dompile serve for live preview and incremental rebuild.

Legacy of Aggregation

Dogpile pioneered the art of peaceful aggregation in web search—querying multiple engines and presenting clean, unified results from diverse sources :contentReference[oaicite:1]{index=1}. DomPile reimagines that ethos—assembling HTML bits into coherent pages, but you remain in full control.
